## DECISION CALIBRATION (read carefully)

Implication problems split roughly 50/50 between TRUE (Eq1 logically forces Eq2) and FALSE (some magma satisfies Eq1 but violates Eq2). DO NOT default to FALSE just because counterexamples are concrete and proofs are abstract. Both verdicts require evidence:

- Output **FALSE** only when you have an explicit verified finite counterexample magma (an operation table where Eq1 holds for all relevant assignments and Eq2 fails on at least one).
- Output **TRUE** when, after a serious search, no counterexample appears across small magmas (sizes 2–4, projection-like, modular, boolean-like families) AND there is at least one plausible algebraic reason for Eq1 to force Eq2 (substitution chain, term-rewriting, structural collapse).
- "I cannot find one" is NOT the same as "none exists" — but if you have honestly tried the standard families below and nothing falsifies Eq2, your prior should shift toward TRUE, not toward a defensive FALSE.

## CHEAT SHEET: EQUATIONAL THEORIES IMPLICATION

### 1. Objective
- Given Eq1 (`L1 = R1`) and Eq2 (`L2 = R2`), decide whether Eq1 implies Eq2 over all magmas.
- Strategy:
  - Run a quick TRUE check (structural filters) AND a quick FALSE check (counterexample search) in parallel.
  - Stop as soon as one branch resolves the question; otherwise commit based on the weight of evidence.

### 2. Core Principles (symmetric)
- A single verified finite counterexample is decisive for FALSE.
- Failure to find a counterexample in standard small families, combined with a plausible substitution argument, is decisive for TRUE.
- Treat both directions as equally likely a priori.

### 3. Fast Structural Filters
Immediate TRUE checks:
- Eq2 tautology: `L2 == R2`.
- Eq2 is Eq1 up to variable renaming or side swap.
- Eq2 is obtainable from Eq1 by direct substitution (e.g. swap a variable for a term that Eq1 makes equal to it).
- Eq1 has very strong collapse or projection behavior forcing trivial structure.

Immediate FALSE check:
- If a size-2 or size-3 magma (projection, constant, boolean-like) satisfies Eq1 and violates Eq2, output FALSE.

### 4. Counterexample Search Ladder
Run in this order, but stop early if Eq1 and Eq2 are clearly related by direct substitution:

1. Size-2 exhaustive — enumerate all binary tables on `{0,1}`.
2. Size-3 targeted families — left-zero, right-zero, constant, projection-like, commutative subsets.
3. Algebraic parametric families — `x*y = a x + b y + c (mod n)` for small primes/moduli.
4. Boolean-like operators — xor / and / or / implication-like tables.
5. Random + local hill-climb — random tables tuned to satisfy Eq1, then search Eq2 violations.

Stop immediately when a verified counterexample appears.

If you finish all five levels without a counterexample, do NOT default to FALSE. Re-examine whether Eq1's terms actually impose enough structure on Eq2's terms (substitution, congruence). If yes, conclude TRUE.

### 5. Validation Protocol
For any candidate counterexample:
1. Verify Eq1 on all assignments of Eq1 variables.
2. If Eq1 passes, check Eq2 on all assignments, or find one failing witness.
3. Report: domain, full operation table, one explicit failing assignment for Eq2, value trace of both sides on that assignment.

Never output FALSE without this full verification.

For TRUE conclusions, sketch the substitution / rewriting argument: which subterm of Eq2 reduces to which side of Eq1.

### 6. Heuristic Prioritization Signals
- Lower Eq1 complexity → easier to satisfy randomly → counterexamples easier to find.
- Low subterm overlap from Eq1 to Eq2 → counterexamples more likely.
- High subterm overlap (Eq2 looks reachable from Eq1 by substitution) → TRUE more likely.
- Eq2 structurally stricter than Eq1 → counterexamples more likely.

### 7. Counterexample-Friendly Candidate Templates
- constant tables, left projection (`x*y=x`), right projection (`x*y=y`), near-projection with one altered cell, affine/modular tables with small modulus.
