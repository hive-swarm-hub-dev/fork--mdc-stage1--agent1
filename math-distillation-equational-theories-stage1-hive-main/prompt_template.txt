## Step 1 — Extract structural features

For each side of `A` and `B`, compute (cheaply, in your head):
- **leaves**: list of variables in left-to-right order
- **leftmost / rightmost** leaf
- **left depth**: number of left-edges from root to the leftmost leaf
- **right depth**: number of right-edges from root to the rightmost leaf
- **vars(E)**: number of distinct variables in equation E
- **dup(E)**: total leaves minus vars(E) (i.e. how many repeated occurrences)

This reduces both equations to a small fixed feature vector.

## Step 2 — TRUE shortcuts (return TRUE on match)

Apply in order. Stop at the first match.

`X1` **Identity**: If both sides of `B` are syntactically the same (same tree), return TRUE.
   - Example: `x*(y*z) = x*(y*z)` is trivially true.

`X2` **Renaming/swap**: If `A` and `B` are the same equation up to a single consistent renaming of variables (and optionally a swap of which side is "left" vs "right"), return TRUE.
   - Example: `A: x = y*x` and `B: a = b*a` are the same up to renaming.

`X3` **Collapsing source**: If one side of `A` is a single variable that does NOT appear on the other side, then `A` forces every magma to collapse to one element, so `A` implies anything. Return TRUE.
   - Example: `A: x = y*z` (LHS has `x`, not in RHS), so `A` forces all elements equal.

`X4` **Substitution**: If you can derive `L_B == R_B` by substituting subterms using `A` (using `L_A = R_A` rewrites), return TRUE. Sketch the rewrite chain in your justification.

---
## Step 3 — Forced-behavior identifications (use to argue TRUE or to find FALSE)

These identify simple operation tables that `A` forces. Once you know `A` forces a specific operation, you can directly check whether `B` holds in that operation.

`F1` If `A` reduces (up to renaming/swap) to `x = x*y` with `x ≠ y`, then `A` forces **left projection**: `a*b = a` for all a,b. Now check whether `B` holds in left projection — if yes, return TRUE; if no, return FALSE.

`F2` If `A` reduces to `x = y*x` with `x ≠ y`, then `A` forces **right projection**: `a*b = b`. Check `B` in right projection.

`F3` If both sides of `A` are products `(s*t)` and `(s*u)` with the same left child `s` but different right children, `A` forces a **left-unary** operation `a*b = f(a)` (depends only on left arg). Check `B` in such a magma.

`F4` Mirror of F3 for right child → forces right-unary `a*b = g(b)`.

## Step 4 — Counterexample search (only if no rule above fires)

Try small magmas to falsify `A → B`. Use ONE concrete table per attempt; verify `A` holds for all assignments and find a `B` violation.

Order: size-2 boolean tables (xor, and, or, projection, constant) → size-3 modular forms (`a*b = (αa+βb+γ) mod 3`) → size-4 specialized tables. Stop at the first verified counterexample.

If no FALSE-returning rule fires, **return TRUE**. Do NOT default to FALSE just because you cannot find a counterexample after limited search; absence of a counterexample combined with absence of obvious algebraic obstruction is evidence FOR TRUE.
